# Jupiter RAG — Document Intelligence Tool

A RAG-powered document intelligence tool built for Jupiter Asset Management. Allows staff to interrogate fund factsheets and regulatory documents in plain English using Claude AI. Every answer is grounded strictly in the document, with source citations, confidence scores, and page references.

## How to run

**First time only:**
```
pip install -r requirements.txt
```

**Every time:**

Double-click `run.bat` — it fetches the latest factsheets, starts the local server, and opens the app automatically.

Or manually:
```
python fetch_factsheets.py
python -m http.server 8000
```
Then open `http://localhost:8000/jupiter-rag.html`.

> The app must be served via a local server. Never open the HTML file directly from the file system — it will not work.

## Requirements

- Python 3
- pip
- An Anthropic API key from [console.anthropic.com](https://console.anthropic.com)

## Features

| Feature | Description |
|---------|-------------|
| **Fund search box** | Reads `funds.json` on load and shows a live search box listing all downloaded factsheets — type to filter, click to load instantly |
| **PDF upload** | Drag and drop any PDF as a fallback or to load your own document |
| **Automated factsheet fetcher** | `fetch_factsheets.py` scrapes jupiteram.com and downloads all public factsheet PDFs to `./factsheets/` |
| **RAG pipeline** | Text extracted page-by-page via PDF.js, chunked at ~420 words with 55-word overlap, scored by TF-IDF at query time |
| **Document health check** | Runs automatically on load — checks for risks, regulatory obligations, performance data, fund manager name, fees, and investment objective |
| **Five team modes** | Each mode has its own system prompt and 20-question bank tailored to that team's priorities |
| **Predictive question bar** | Fuzzy-match autocomplete across the active team's question bank — or type any free-form question |
| **Confidence indicator** | High / Medium / Low badge on every answer based on retrieval relevance scores |
| **Source citations** | Every answer cites the source section and page number from the document |
| **Comparison mode** | Load two documents simultaneously — questions draw from both, sources labelled Document 1 / Document 2 |
| **RAG memory panel** | Collapsible sidebar showing exactly which chunks were sent to Claude for each answer |
| **Briefing pack** | Compiles the full conversation into formatted text ready to copy into an email or Word document |
| **Client letter drafting** | Generates a complete investor update letter in retail, institutional, or wholesale tone |
| **Estimated time saved** | Each answer shows a rough calculation of manual research time saved |
| **Copy buttons** | One-click copy on every AI answer |
| **Mobile responsive** | Three-tab layout on mobile: Docs / Chat / Sources |

## Team modes

| Team | Focus |
|------|-------|
| **Investment** | Holdings, performance, risk, positioning, benchmark comparison |
| **Compliance** | FCA obligations, UCITS compliance, risk disclosures, restrictions |
| **Marketing** | Selling points, plain-English summaries, investor communications |
| **Client Relations** | Investor-facing explanations, share classes, dealing terms |
| **Operations** | ISINs, deadlines, service providers, fees, settlement terms |

## Project structure

```
Jupiter-RAG/
├── jupiter-rag.html          # The app — single self-contained HTML file
├── fetch_factsheets.py       # Scrapes and downloads Jupiter fund factsheets
├── requirements.txt          # Python dependencies
├── run.bat                   # One-click launcher (fetch → server → browser)
├── funds.json                # Generated by fetch_factsheets.py (gitignored)
├── factsheets/               # Downloaded PDFs (gitignored)
├── SETUP.md                  # Step-by-step setup guide
├── CHANGELOG.md              # Session history
└── NOTES.md                  # Known issues and future ideas
```

## Built with

- [Anthropic Claude API](https://anthropic.com) — `claude-sonnet-4-20250514`
- [PDF.js](https://mozilla.github.io/pdf.js/) — PDF text extraction
- [Python requests](https://requests.readthedocs.io) + [BeautifulSoup4](https://www.crummy.com/software/BeautifulSoup/) — factsheet scraping
- Vanilla JavaScript — no frameworks, no build step
